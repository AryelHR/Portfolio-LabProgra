## Part 2 — Target Profile and Justification

### Common patterns identified across the five postings

After reviewing all five job descriptions, several requirements appeared consistently:

- Solid HTML5 and CSS3 are non-negotiable in every single posting. This is the foundation everything else is built on.
- JavaScript is required at intermediate or higher level in four out of five postings.
- Figma and the ability to translate designs into code appear in three postings directly, and are implied in the rest.
- Responsive design and cross-device compatibility are expected in every role without exception.
- Git and version control are assumed as a baseline, not listed as a differentiator.
- Design Systems appear in two postings explicitly, and component consistency is expected everywhere.
- Soft skills like autonomy, attention to detail, and clear communication are mentioned across the board.

The jobs ranged from CMS-focused maintenance roles to component-driven application development and SEO-integrated builds. Despite the range, the core frontend fundamentals remained the same throughout.

### Target professional profile

**Objective role: Junior Frontend Developer focused on UI implementation and e-commerce**

Based on the analysis, the target profile is a frontend developer who can take a Figma design and build it accurately — someone who produces clean, responsive, and consistent interfaces using HTML5, CSS3, and JavaScript. The profile leans toward the type of role at I2B Technologies and Simera: frontend implementation in commercial or B2C environments, where precision, design fidelity, and practical autonomy are the core expectations.

This profile was chosen because it aligns with the skills currently being developed — HTML5 and CSS3 at a strong level, JavaScript growing, and a genuine interest in the UI side of development rather than backend logic.

### Justification of the site

The portfolio was designed as if applying to a junior frontend role. Every section maps back to something that appeared in the job research:

- The projects section demonstrates Figma-to-code implementation, Design System application, and responsive e-commerce UI — skills directly requested in Job 1 and Job 2.
- The skills section reflects the tech stack that appeared most consistently across all five postings.
- The site itself is built without frameworks (pure HTML5 and CSS3), which proves the fundamentals are solid rather than hidden behind a library.
- The "About" section communicates availability, location, and a clear professional focus — which matters in remote-first roles like Jobs 2, 4, and 5.

### Key differentiators

1. The portfolio was built without any CSS framework or template. Every layout, animation, and component was written from scratch, which demonstrates genuine mastery of the fundamentals the market consistently asks for.
2. The projects were selected and built specifically to reflect what hiring teams in B2C and e-commerce environments look for: landing pages, product catalogs with interactive filters, and a complete Design System documentation — not generic to-do apps.

## Part 4 — Project Structure and Technologies

### Technologies used

- HTML5 — semantic structure throughout: `nav`, `section`, `footer`, `h1`–`h3`, `p`, `a`, `img` with alt attributes
- CSS3 — custom properties, Flexbox, CSS Grid, media queries, transitions, and animations
- JavaScript — used only for the mobile hamburger menu toggle via a CSS checkbox pattern

No frameworks, no templates, no Bootstrap, no Tailwind.

### File structure

```
/
├── index.html
├── proyecto1.html
├── proyecto2.html
├── proyecto3.html
├── css/
│   ├── styles.css
│   ├── responsive.css
│   └── components.css
└── assets/
    ├── icons/
    └── images/
        └── projects/
```

### Sections in the site

- **Hero** — introduction, role, description, CTA buttons, social links, and tech badges
- **About** — personal summary, professional focus, stats (projects, technologies, commits)
- **Skills** — categorized by Frontend, Tools, and Currently Learning, with visual skill level indicators
- **Projects** — three featured projects: Maison Claude (e-commerce landing), NORR Studio (product catalog with filters), and Volta (Design System documentation)
- **Contact** — email and social links
